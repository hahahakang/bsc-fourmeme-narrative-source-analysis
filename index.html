BSC Four.meme 叙事源追踪与算法模型 BSC Four.meme Narrative Source & Signal Model

这是从原链上复盘拆出来的新项目：不覆盖原文章，专门追踪“故事最早从哪里来”、谁是强相关消息源，以及买入后不同发酵窗口的盈利概率。 This is a separate project split from the original onchain report. It tracks where each story started, which sources are strongly related to the token, and which post-entry fermentation windows historically worked better.

每个样本都尽量拆成“源头实体、源头事件、买入相对源头的时间差、强相关程度、机器人意义”。没有原始链接就降级处理。

Each sample is broken into source entity, source event, buy-vs-source timing, relationship strength, and bot implication. Missing original links are downgraded.

3. 模型先提醒，不自动梭哈3. Alert First, Automate Later

当前目标是沉淀信号规则：先做信息雷达和人工确认，再讨论半自动执行。样本库足够大之前，不把单案例包装成确定性。

The current goal is a signal rulebook: build radar and human confirmation first, then discuss semi-automation. Single cases are not treated as certainty.

这轮新增了什么What This Version Adds

上一个版本回答“这个交易员到底在哪个 FDV 节点赚钱”。这个版本继续往前追：他买的 token 背后有没有源头，源头是人物、公司、项目、官方功能，还是纯社区 meme？

目前源头字段采用“可追踪候选”口径：链上事实是已计算数据；叙事来源是待核验证据层。没有原始链接的，不会伪装成结论。

The previous version answered where the trader made money by FDV bucket. This version moves upstream: did each token have a source, and was the source a person, company, project, official feature, or community meme?

The source fields are tracked as evidence candidates. Onchain facts are calculated data; narrative origins are an evidence layer. Missing original links are not presented as conclusions.

模型方向Model Direction

先让机器人做信息雷达：发现源头、抓第一批 token、计算 FDV/流动性/成交窗口、判断 leader/copycat，再推送人工确认。自动买入必须等样本库足够大。

时间优势是核心变量：源头出现、token 部署、池子形成、交易员买入、大号放大、价格突破，每个时间差都要记录。

The first bot should be an information radar: detect sources, capture the first token batch, calculate FDV/liquidity/trading windows, score leader versus copycat, and then ask for human confirmation.

Time advantage is the key variable: source publication, token deployment, pool formation, trader entry, influencer amplification, and price breakout all need timestamps.

证据等级Evidence Grades

  • A：官方/创始人/公司原始链接，且早于 token 爆发。
  • B：早期解释账号，早于大面积传播。
  • C：快速部署者或平台事件。
  • D：大号放大，更多是卖出/避险参考。
  • E：无链接、截图、转述，暂不入模型。
  • A: original official/founder/company link before token breakout.
  • B: early explanation account before broad spread.
  • C: fast deployer or platform-event attribution.
  • D: influencer amplification, more useful for exit/risk control.
  • E: no link, screenshot, or verifiable source; excluded from the model for now.

风控边界Risk Boundaries

  • 不要从单案例直接全自动买。
  • 跳过高 FDV 追高、薄池、疑似貔貅、伪官方。
  • 记录每次命中和误报，source library 比单次收益更重要。
  • 先提醒、再半自动、最后才讨论自动执行。
  • Do not turn one case into a fully automated buy rule.
  • Skip high-FDV chasing, thin pools, honeypot-like tokens, and fake official names.
  • Record every hit and false positive; the source library matters more than one trade.
  • Alert first, then semi-automate, and only later discuss full execution.
